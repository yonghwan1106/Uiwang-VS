### "의왕 VS 정책 랩" 포털 구축 PRD (Product Requirements Document)

본 문서는 '의왕 VS 정책 랩' 웹 애플리케이션(이하 'VS-Lab') 구축을 위한 제품 요구사항을 정의합니다.

**1. 개요 (Overview)**

  * **1.1. 제품명:** 의왕 VS(Verbal Sampling) 정책 랩 (Internal Project Name: Uiwang-VS-Lab)
  * **1.2. 목표:** 의왕시 정책혁신팀 [1] 및 지정 공무원들이 LLM의 '버벌라이즈드 샘플링(VS)' [1] 기법을 활용하여 창의적이고 다양한 정책 아이디어를 발굴, 아카이빙, 공유할 수 있는 내부용(Internal) 웹 애플리케이션을 구축한다.
  * **1.3. 핵심 사용자 (User Personas):**
      * **정책 담당자 (공무원):** (예: 기획예산과 정책혁신팀 [1]) 신규 사업 [2, 5], 공모전 아이디어 검증 [1], 현안(예: 불법 주정차 [14, 15]) 해결을 위한 창의적 대안이 필요한 공무원.
      * **관리자 (TFT 리더):** 시스템 사용 통계, 우수 아이디어 아카이빙 승인, 사용자 계정 관리를 담당.

**2. 기술 스택 (Tech Stack) - 확정**

  * **2.1. Frontend:** Next.js (v14 이상, App Router), TypeScript, Tailwind CSS
  * **2.2. Deployment:** Vercel (GitHub 비공개 저장소와 연동)
  * **2.3. Database:** Google Sheets (아이디어 로그, 프롬프트 템플릿, 사용자 피드백 저장용)
  * **2.4. Core AI API:** Anthropic Claude Sonnet 4.0 API (사용자 지정)
  * **2.5. Maps API:** Naver Maps API (사용자 지정)
  * **2.6. 인증 (Authentication):** NextAuth.js (v5 이상) - 의왕시 내부 Google Workspace 계정 연동 (Google OAuth Provider)

**3. 주요 기능 (Features) - User Stories**

  * **Epic 1: 인증 및 보안 (Authentication & Security)**

      * **1.1 (로그인):** 정책 담당자는 의왕시 Google Workspace 계정(@uiwang.go.kr 등)으로만 로그인할 수 있다. (NextAuth.js Google Provider의 `signIn` 콜백에서 `profile.email_verified` 및 도메인 검증)
      * **1.2 (보안):** 모든 API 키 (Claude, Google Sheets, Naver) 및 인증 시크릿은 Vercel 환경 변수에 'Sensitive'로 저장한다.
      * **1.3 (데이터 처리):** 모든 API 호출(Claude, Google Sheets)은 Next.js App Router의 'Server Actions' 또는 'Route Handlers' (API Routes)를 통해 서버 측에서만 처리되어 클라이언트에 키가 노출되지 않도록 한다.

  * **Epic 2: VS 프롬프트 생성기 (VS Prompt Generator) - 핵심 기능**

      * **2.1 (인터페이스):** 사용자가 '정책 현안'을 입력할 수 있는 메인 텍스트 에어리어(Textarea)를 제공한다.
      * **2.2 (VS 프롬프트 적용):** 사용자가 '창의성 레벨' (예: 낮음 - 0.3, 중간 - 0.1, 높음 - 0.05)을 선택할 수 있는 슬라이더나 버튼을 제공한다. '생성하기' 버튼 클릭 시, 백엔드(Server Action)는 이 값을 받아 Claude Sonnet 4.0 API에 '시스템 프롬프트' [1] 방식으로 VS 프롬프트를 전송한다.
          * *예시 시스템 프롬프트:* `"당신은 의왕시 정책혁신팀을 돕는 AI 어시스턴트입니다. 다음 현안에 대해, 확률 ${창의성_레벨} 이하의 창의적이고 실현 가능한 정책 대안 5가지를 각 예상 확률과 함께 생성하십시오." [1]`
      * **2.3 (결과 출력):** AI가 생성한 '창의적 대안 목록'과 '각 확률' [1]을 구조화된 UI(카드 형태)로 스트리밍하여 표시한다.
      * **2.4 (프롬프트 템플릿):** 자주 사용하는 VS 프롬프트(예: '시민제안 개선', '타 지자체 융합', '신규사업 발굴' [2, 5]) 템플릿을 드롭다운에서 선택할 수 있다. (템플릿 목록은 Google Sheets에서 관리)

  * **Epic 3: 아이디어 아카이빙 (Database Integration)**

      * **3.1 (저장):** 사용자는 생성된 VS 결과물 중 '우수 아이디어'를 선택하여 '아카이브 저장' 버튼을 누를 수 있다.
      * **3.2 (Google Sheets 연동):** '저장' 시, Next.js Server Action이 `googleapis` 라이브러리를 통해 Google Sheets API를 호출한다. 지정된 Google Sheet의 '아이디어\_DB' 시트에 (ID, 날짜, 담당자, 원본 현안, VS 아이디어, 확률, 키워드) 데이터 행(row)을 추가한다.
      * **3.3 (아카이브 조회):** '아카이브' 페이지에서 Google Sheet에 저장된 모든 아이디어를 실시간으로 불러와 검색하고 필터링(날짜, 키워드, 담당자)하여 조회할 수 있다.

  * **Epic 4: (선택) 현안 시각화 (Naver Maps API Integration)**

      * **4.1 (지도 연동):** '현안 입력' 페이지에서 '지역 기반 현안' 옵션을 체크하면 Naver Maps API가 로드되어 의왕시 지도가 표시된다.
      * **4.2 (데이터 매핑 제안):** AI가 VS 아이디어를 생성할 때, '지도 시각화가 필요한 아이디어'(예: '특정 교차로에 감응 신호 설치')가 있다면, 이를 Naver Maps API와 연동하여 지도 위에 마커로 표시하는 방안을 제안한다. (예: "제안3: OO동 교차로에 감응 신호 설치 [지도에 표시]")

**4. 아키텍처 및 배포 (Architecture & Deployment)**

  * **4.1. Google Cloud 설정:**
      * Google Cloud Platform에서 새 프로젝트를 생성한다.
      * 'Google Sheets API' 및 'Google Drive API'를 활성화한다.
      * 'IAM 및 관리자' \> '서비스 계정'에서 새 서비스 계정을 생성한다.
      * 생성된 서비스 계정에 'Editor' 역할을 부여하고, '키' 탭에서 JSON 키 파일을 생성 및 다운로드한다.
  * **4.2. Google Sheets 설정:**
      * 데이터베이스로 사용할 Google Sheet를 생성한다. (예: '의왕-VS-Lab-DB')
      * 시트 (Tabs) 구성: `아이디어_DB`, `프롬프트_템플릿`, `사용자_로그`
      * 우측 상단 '공유' 버튼을 눌러, 다운로드한 JSON 키 파일 안의 `client_email` (예:...@...iam.gserviceaccount.com)을 추가하고 '편집자' 권한을 부여한다.
  * **4.3. Vercel 배포:**
      * GitHub에 비공개(Private) 저장소를 생성하고 Next.js 프로젝트를 Push한다.
      * Vercel 프로젝트를 생성하고 GitHub 저장소를 연동한다.
      * Vercel 프로젝트 'Settings' \> 'Environment Variables'에 다음 키를 등록한다. (모두 'Sensitive' 타입, 클라이언트 노출 방지):
          * `CLAUDE_API_KEY`: Anthropic API 키.
          * `GOOGLE_SHEETS_CLIENT_EMAIL`: 서비스 계정의 `client_email`.
          * `GOOGLE_SHEETS_PRIVATE_KEY`: 서비스 계정의 `private_key` (JSON 파일의 `-----BEGIN PRIVATE KEY...` 내용을 \\n 포함하여 그대로 복사).
          * `GOOGLE_SHEET_ID`: 대상 Google Sheet의 URL에서 추출한 ID.
          * `NAVER_MAPS_CLIENT_ID`: (필요시) 네이버 맵 API 클라이언트 ID.
          * `NEXTAUTH_SECRET`: NextAuth.js용 비밀 키 (OpenSSL 등으로 생성).
          * `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`: Google Cloud OAuth 2.0 클라이언트 ID 및 보안 비밀.

**5. 기타 제안 API (Suggested APIs for Future Expansion)**

  * **5.1. 의왕시 공공데이터 API (제안):**
      * *내용:* 현재 의왕시는 공공데이터포털에 표준데이터셋이 0건 [29]으로 확인됩니다. 향후 '불법 주정차 단속 현황' [14, 15], 'ITS 교통량 데이터' [16, 17] 등이 API로 제공된다면, VS 프롬프트에 실시간 데이터를 '근거 자료(Context)'로 함께 제공하여 "데이터 기반의 창의적 아이디어"를 도출할 수 있습니다.
  * **5.2. Google Drive API (문서 자동화):**
      * *내용:* Google Sheets에 아카이빙된 우수 아이디어를 선택하여 '보고서 생성' 버튼을 누르면, Google Drive API가 'Google Docs' 템플릿에 내용을 자동으로 채워 넣어 공식 보고서 초안을 생성하는 기능을 제안합니다.

**6. 비기능적 요구사항 (Non-Functional Requirements)**

  * **6.1. 보안:** 내부망 접속을 원칙으로 하거나, NextAuth.js를 통한 Google Workspace 조직 인증을 필수로 한다.
  * **6.2. 성능:** Claude Sonnet 4.0 API 응답 시간을 10초 이내로 유지하기 위해 스트리밍 응답(Streaming Responses)을 적용한다.
  * **6.3. 확장성:** 향후 의왕시 전 부서로 확대 사용을 대비, Google Sheets 대신 전문 DB(예: Vercel Postgres)로 마이그레이션이 용이하도록 데이터 접근 로직을 모듈화하여 설계한다.
